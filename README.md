## 📌 Overview

**CareerMatch AI** is a production-ready career recommendation app that takes a user's skills — typed manually or extracted from an uploaded CV — and ranks the most relevant job roles from a database using **semantic embedding similarity**.

Unlike keyword-matching approaches, this system understands that *"data wrangling"* and *"data manipulation"* are the same thing, and that a *"machine learning engineer"* needs skills similar to a *"data scientist."* This is possible because both the user's skills and every job's required skill set are encoded into the same high-dimensional semantic space using `all-MiniLM-L6-v2` from Sentence Transformers.

## 🧠 How It Works

```
┌─────────────────────────────────────────────────────────────┐
│                      COLAB NOTEBOOK                          │
│                                                             │
│  job_skill_set.csv  ──►  Clean & normalise skills           │
│                      ──►  Encode with all-MiniLM-L6-v2      │
│                      ──►  job_embeddings.npy   (N × 384)    │
│                      ──►  job_titles_skills.csv              │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      STREAMLIT APP                           │
│                                                             │
│  User inputs skills / uploads CV                            │
│         │                                                   │
│         ▼                                                   │
│  Encode user skills with same model  ──►  (1 × 384)        │
│         │                                                   │
│         ▼                                                   │
│  Cosine similarity  =  job_embeddings @ user_emb.T          │
│         │                                                   │
│         ▼                                                   │
│  Rank & display top-N matches with scores & skill chips     │
└─────────────────────────────────────────────────────────────┘
```

### Why cosine similarity?

Both job embeddings and user embeddings are **L2-normalised** during encoding (`normalize_embeddings=True`). This means cosine similarity reduces to a simple dot product — making ranking across thousands of jobs near-instant with NumPy.

## 🚀 Getting Started

### Prerequisites

- Python 3.10 or higher
- A Google account (for Colab + Drive)
- Your `job_skill_set.csv` dataset with these columns:

  | Column | Description |
  |---|---|
  | `job_id` | Unique identifier for each role |
  | `category` | Job category / department |
  | `job_title` | Name of the role |
  | `job_description` | Free-text description |
  | `job_skill_set` | Comma-separated list of required skills |

---

### Step 1 — Generate Embeddings (Google Colab)

1. Upload `career_recommender_colab.ipynb` to [Google Colab](https://colab.research.google.com)
2. Place your `job_skill_set.csv` in Google Drive at:
   ```
   MyDrive/career_recommender/job_skill_set.csv
   ```
3. Run all cells. The notebook will:
   - Validate your CSV columns
   - Clean and normalise the `job_skill_set` column
   - Encode all job skill sets using `all-MiniLM-L6-v2`
   - Save `job_embeddings.npy` and `job_titles_skills.csv` to Drive
   - Offer a direct download via `files.download()`

4. Download both output files.

---

### Step 2 — Set Up Locally

Clone or download this repository, then set up a virtual environment:

```bash
# Create and activate a virtual environment
python -m venv .venv

# macOS / Linux
source .venv/bin/activate

# Windows
.venv\Scripts\activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

Place the Colab output files in the `data/` folder:

```bash
mkdir data
mv path/to/job_embeddings.npy    data/
mv path/to/job_titles_skills.csv data/
```

---

### Step 3 — Run the App

```bash
streamlit run app.py
```

Open your browser at `http://localhost:8501`. The model and data are loaded once and cached — subsequent interactions are instant.

### Skill Input — Two Methods

**Manual entry**
Type your skills as a comma-separated list in the text area:

All runtime configuration lives in `app.py` at the top of the file:

DATA_DIR        = Path(__file__).parent / "data"
EMBEDDINGS_PATH = DATA_DIR / "job_embeddings.npy"
SLIM_CSV_PATH   = DATA_DIR / "job_titles_skills.csv"

To use a different Sentence Transformer model (e.g. `all-mpnet-base-v2` for higher accuracy at the cost of speed), change `MODEL_NAME` and re-run the Colab notebook to regenerate embeddings with the new model.

## 🧹 Data Cleaning Logic

Both the Colab notebook and `app.py` use the same normalisation pipeline so that user input and job embeddings live in a consistent space:

```python
def clean_skills(raw: str) -> str:
    raw = raw.lower()
    skills = re.split(r"[,;|/\n]+", raw)
    cleaned = []
    for skill in skills:
        skill = re.sub(r"[^a-z0-9 +#.\-]", " ", skill)
        skill = re.sub(r"\s+", " ", skill).strip()
        if skill:
            cleaned.append(skill)
    return ", ".join(cleaned)
```

- Lowercases everything
- Splits on commas, semicolons, pipes, slashes, and newlines
- Strips special characters (preserving `+`, `#`, `.`, `-` for skills like `C++`, `C#`, `.NET`)
- Collapses whitespace
- Drops empty tokens

## 🔧 Troubleshooting

**`job_embeddings.npy` not found**
Make sure you have run the Colab notebook and placed both output files inside the `data/` directory.

**PDF text extraction returns nothing**
Some PDFs are image-based (scanned documents). `pdfplumber` only works on text-layer PDFs. For scanned CVs, use an OCR tool like `pytesseract` to convert to text first, then use the TXT upload option.

**Model download is slow on first run**
The `all-MiniLM-L6-v2` model (~90 MB) is downloaded from Hugging Face on first use and cached locally. Subsequent runs load from cache instantly.

**Mismatch between embedding rows and CSV rows**
This happens if you edit the CSV after generating embeddings. Re-run the Colab notebook to regenerate both files together from the same source data.

**Category filter shows no results**
If filtering by a specific category returns nothing, your CSV's `category` column may have inconsistent casing or whitespace. The Colab notebook cleans `job_skill_set` but not `category` — check for trailing spaces or mixed case in that column.
